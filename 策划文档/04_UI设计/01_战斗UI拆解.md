# 战斗界面 UI 拆解文档

> 版本：v1.0  
> 目标：基于当前线框图，拆解战斗界面的区域结构、UI 组件、信息层级与交互逻辑，作为后续视觉设计、原型制作和前端实现的基础说明。

---

## 1. 界面定位

该界面属于 **防线守卫 + 角色战斗 + 手牌技能** 的混合型战斗界面。

核心体验不是传统纯塔防，也不是纯卡牌，而是：

```text
敌人从上方推进
↓
玩家角色 / 城墙在下方防守
↓
玩家通过底部手牌释放技能
↓
技能影响战场敌人
↓
击杀敌人获得经验、能量或资源
↓
波次持续推进，直到完成关卡
```

关键词：

- 竖屏移动端
- 实时防守
- 波次推进
- 局内成长
- 人物核心
- 手牌释放技能
- 城墙 / 基地血量作为失败条件

---

## 2. 整体区域划分

当前界面可拆分为 5 个主区域：

| 区域 | 位置 | 主要作用 | 信息优先级 |
|---|---|---|---|
| 顶部状态栏 | 屏幕最上方 | 暂停、时间、关卡、波次 | 中 |
| 战斗场地 | 中上大面积区域 | 敌人、弹道、技能、伤害表现 | 高 |
| 城墙 / 防线区 | 战斗场地下方 | 表示防守边界、承接人物站位 | 高 |
| 角色状态区 | 城墙下方 | 人物、姓名、城墙血条、人物能量 | 高 |
| 手牌操作区 | 屏幕底部 | 当前手牌、能量、弃牌、牌堆 | 高 |

整体结构如下：

```text
┌────────────────────────────┐
│ 顶部状态栏：暂停 / 时间 / 关卡 / 波次 │
├────────────────────────────┤
│ 局内等级 / 经验条                     │
├────────────────────────────┤
│                                      │
│              战斗场地                 │
│   敌人 / 弹道 / 技能范围 / 伤害数字    │
│                                      │
├────────────────────────────┤
│                城墙                   │
├────────────────────────────┤
│ 城墙血条 / 人物 / 人物姓名             │
│ 当前能量 / 人物能量                   │
├────────────────────────────┤
│ 弃牌堆     扇形手牌区       牌堆       │
└────────────────────────────┘
```

## 3. 顶部状态栏拆解

顶部状态栏承载全局战斗信息，不应抢占战斗区域视觉权重。

### 3.1 暂停按钮

线框标注：

```text
暂停
```

位置：左上角。

功能：

- 暂停战斗
- 打开暂停菜单
- 进入设置、退出、重开、继续等二级操作

建议表现：

| 项目 | 建议 |
|---|---|
| 形式 | 小型圆角按钮 |
| 内容 | 图标优先，例如 `Ⅱ` |
| 文本 | 可保留，但正式 UI 中建议弱化 |
| 状态 | 点击后打开暂停弹窗 |

---

### 3.2 对局时间

线框标注：

```text
对局时间
```

位置：顶部左侧，暂停按钮右侧。

功能：

- 显示当前战斗已进行时间
- 支持限时关卡、结算评分、成就判断

建议格式：

```text
01:35
```

或：

```text
时间 01:35
```

---

### 3.3 关卡信息

线框标注：

```text
关卡·关卡名
```

位置：顶部居中。

功能：

- 告诉玩家当前所在关卡
- 作为战斗场景标题
- 强化关卡主题和当前进度感

建议正式文案：

```text
1-1 教堂广场
```

或：

```text
关卡 1-1
教堂广场
```

建议：关卡名应是顶部状态栏中视觉权重最高的信息之一，但不要过大，以免压缩战斗场地。

---

### 3.4 波次信息

线框标注：

```text
波次：1/20
```

位置：顶部右侧。

功能：

- 显示当前波次进度
- 表示战斗压力和关卡剩余长度
- 是防守玩法中的核心反馈信息

建议正式文案：

```text
第 1 波 / 20
```

或：

```text
波次 1/20
```

可扩展内容：

| 扩展项 | 示例 |
|---|---|
| 下一波倒计时 | 下一波 08s |
| 当前波剩余敌人 | 敌人 6/12 |
| Boss 波提示 | Boss 即将出现 |

---

## 4. 局内等级 / 经验条

线框标注：

```text
局内等级(经验条)
```

位置：顶部状态栏下方，横向长条。

功能：

- 表示本局内成长系统
- 击杀敌人获得经验
- 经验满后触发升级奖励、抽卡、强化或天赋选择

组件拆解：

| 组件 | 作用 |
|---|---|
| 当前等级 | 显示局内等级，例如 Lv.3 |
| 经验进度条 | 显示距离下一级的进度 |
| 经验数值 | 可选，例如 35/100 |
| 升级提示 | 经验满时提示可升级 |

建议表现：

```text
Lv.3  ███████░░░  65/100
```

或：

```text
局内等级 Lv.3
```

注意事项：

- 该信息重要，但不是玩家每秒都需要操作的核心。
- 建议使用细长条，避免占据过多高度。
- 升级时可短暂高亮、闪烁或弹出升级选择。

---

## 5. 战斗场地区域

战斗场地是界面中最大区域，承担战斗表现和敌人推进。

### 5.1 区域功能

主要承载：

- 敌人生成
- 敌人移动路径
- 角色普通攻击
- 技能弹道
- 技能范围
- 伤害数字
- 状态效果
- 掉落物表现
- 精英怪 / Boss 提示
- 危险预警

---

### 5.2 敌人推进结构

敌人通常从上方进入，向下方城墙移动。

```text
敌人出生区
↓
敌人行进区
↓
城墙接触区
```

如果采用三路防守，可以隐性划分为：

```text
左路 | 中路 | 右路
```

建议：

- 路线分割不要过重。
- 可通过地面纹理、道路宽度、敌人站位表达路径。
- 敌人越接近城墙，危险感越强。

---

### 5.3 敌人 UI 元素

敌人身上可包含以下元素：

| 元素 | 说明 |
|---|---|
| 血条 | 显示当前生命值 |
| 精英标签 | 例如 `ELITE` |
| Boss 标签 | Boss 波使用 |
| 状态图标 | 冰冻、燃烧、中毒、减速等 |
| 危险标记 | 接近城墙时显示感叹号 |
| 伤害数字 | 受击时短暂浮现 |

示例：

```text
ELITE
████░░
哥布林队长
```

---

### 5.4 攻击 / 弹道表现

战斗场地需要展示角色和技能的攻击反馈。

| 攻击类型 | 表现方式 |
|---|---|
| 普通攻击 | 子弹、箭矢、光束、飞弹 |
| 穿透攻击 | 直线轨迹 |
| 抛物线攻击 | 曲线轨迹 |
| 范围技能 | 落点圈、爆炸圈、扩散圈 |
| 自动锁定 | 准星、连线、目标框 |
| 延迟释放 | 倒计时圈、预警范围 |

建议保留类似：

```text
推荐落雷点
```

这种战斗辅助提示，但数量应控制，避免遮挡战场。

---

### 5.5 伤害与状态反馈

战斗中可短暂显示：

```text
-128
暴击 560
减速
冰冻
中毒
燃烧
```

规则建议：

- 普通伤害数字短暂出现后淡出。
- 暴击数字更大、更亮。
- 控制类状态应显示小图标，不建议长期显示大文字。
- 精英怪状态信息应比普通怪更明显。

---

## 6. 枪械子弹数组件

线框标注：

```text
枪械子弹数
```

位置：战斗场地右上区域。

功能：

- 显示当前武器弹药
- 反馈换弹状态
- 可能影响普通攻击节奏
- 可能与部分卡牌效果联动

组件拆解：

| 元素 | 说明 |
|---|---|
| 当前弹药 | 当前剩余子弹 |
| 最大弹药 | 弹匣容量 |
| 换弹状态 | 换弹进度 / 冷却 |
| 弹药类型 | 普通弹、穿透弹、冰雷弹等 |

建议表现：

```text
弹药 12 / 30
```

或：

```text
▮▮▮▮▮▯▯▯
```

注意：

- 如果弹药与主角强绑定，可以靠近人物状态区。
- 如果弹药影响战场攻击节奏，可以放在战斗场地边缘。
- 不建议与手牌能量混淆。

---

## 7. 城墙 / 防线区

线框标注：

```text
城墙
```

位置：战斗场地下方，横向分割区域。

功能：

- 表示敌人突破前的最终防线
- 是战斗场地与底部操作区之间的视觉分界
- 承载人物站位、防御设施、炮台、城门等元素
- 受到敌人攻击时产生受击反馈

组件拆解：

| 元素 | 作用 |
|---|---|
| 城墙本体 | 明确防线位置 |
| 中央平台 | 主角站位 |
| 炮台 / 机关 | 可选防御设施 |
| 城门 / 入口 | 敌人突破目标 |
| 受击效果 | 裂纹、震动、火花、红光 |
| 突破警告 | 敌人接近时提示 |

建议视觉方向：

```text
石墙 / 教堂台阶 / 金属防线 / 防御平台
```

该区域需要比线框中更有存在感，因为它承担“失败边界”的认知功能。

---

## 8. 城墙血条

线框标注：

```text
城墙血条
```

位置：城墙下方。

功能：

- 显示基地或城墙耐久
- 代表失败条件
- 血量降低时制造压力反馈

组件拆解：

| 元素 | 说明 |
|---|---|
| 文案 | 基地 HP / 城墙 HP / 城墙耐久 |
| 血条 | 当前生命值 |
| 数值 | 例如 860/1000 |
| 百分比 | 例如 86% |
| 危险状态 | 低血量红色闪烁 |
| 受击反馈 | 血条震动、扣血动画 |

建议表现：

```text
基地 HP  ███████░░░  860 / 1000
```

或：

```text
城墙耐久 86%
```

优先级：高。玩家必须能快速判断是否接近失败。

---

## 9. 人物区域

线框标注：

```text
人物
人物姓名
```

位置：城墙血条下方，中间区域。

### 9.1 人物实体 / 头像

当前线框用圆形表示人物。

功能：

- 表示当前主控角色
- 表示防线核心单位
- 承载角色状态、技能充能、受击反馈

建议设计规则：

| 场景 | 建议 |
|---|---|
| 战场中已有角色实体 | 底部只保留头像与状态 |
| 战场中没有角色实体 | 此区域需要承担主要人物展示 |
| 角色可切换 | 加入头像列表或切换按钮 |
| 角色可释放大招 | 加入大招充能反馈 |

结合当前方向，建议：

```text
战斗场地城墙上方：放真实人物小人 / 立绘
底部人物区域：放头像、姓名、状态、能量
```

这样能避免人物在界面中重复表达。

---

### 9.2 人物姓名

功能：

- 显示当前角色名称
- 可附带职业、等级、星级、武器类型

建议文案：

```text
艾琳
Lv.2 枪械师
```

或：

```text
指挥官 · 艾琳
```

---

## 10. 当前能量

线框标注：

```text
当前能量
```

位置：底部左侧，靠近手牌区。

功能：

- 显示当前可用于打牌的资源
- 玩家出牌前需要优先查看
- 是卡牌玩法中的核心操作资源

建议表现：

```text
⚡ 3 / 5
```

或：

```text
能量 3
```

视觉建议：

| 项目 | 建议 |
|---|---|
| 形态 | 圆形能量球 / 徽章 |
| 位置 | 手牌区左侧或中心偏左 |
| 状态 | 能量不足时相关卡牌变灰 |
| 动画 | 回合开始 / 能量回复时跳动或发光 |

注意：当前能量应与“人物能量”区分。

---

## 11. 人物能量条

线框标注：

```text
人物能量
```

位置：人物下方，横向长条。

需要明确它与“当前能量”的关系。

### 方案 A：人物能量 = 大招能量

推荐方案。

```text
当前能量：用于打牌
人物能量：用于释放角色大招
```

示例：

```text
大招充能 70%
```

优点：

- 两套资源逻辑清晰
- 打牌资源与角色技能不会混淆
- 人物成长和操作反馈更强

---

### 方案 B：人物能量 = 当前出牌能量

不推荐。

如果人物能量就是出牌能量，则不应同时显示“当前能量”和“人物能量”，否则信息重复。

---

### 建议结论

推荐使用双资源结构：

| 资源 | 用途 |
|---|---|
| 当前能量 | 打牌消耗 |
| 人物能量 | 大招 / 角色主动技充能 |

---

## 12. 手牌操作区

线框底部的卡牌区是当前界面的核心操作区。

形式应参考《杀戮尖塔》《吸血鬼爬行者》这类卡牌手牌呈现：

```text
左侧卡牌向左倾斜
中间卡牌正立并略高
右侧卡牌向右倾斜
整体形成扇形手牌
```

不建议使用传统塔防技能栏式的四宫格或固定卡槽。

---

### 12.1 手牌区功能

- 展示当前可用卡牌
- 通过点击或拖拽释放技能
- 显示卡牌费用
- 显示卡牌效果
- 表达卡牌是否可使用
- 支持目标选择、范围选择、路径选择
- 使用后进入弃牌堆

---

### 12.2 单张卡牌组件拆解

| 元素 | 位置 | 说明 |
|---|---|---|
| 费用 | 左上角 | 出牌消耗，最高优先级 |
| 卡牌名称 | 中下部 | 技能 / 道具 / 攻击名称 |
| 卡牌插画 | 中间 | 表现技能内容 |
| 效果描述 | 底部 | 简短说明效果 |
| 类型标签 | 底部或角标 | 攻击、防御、召唤、陷阱等 |
| 稀有度边框 | 整体边框 | 普通、稀有、史诗等 |
| 升级标记 | 名称旁或角标 | `+`、Lv.2、强化 |
| 可用状态 | 整体亮度 | 能量足够则高亮，不足则灰化 |

示例卡牌结构：

```text
┌──────────────┐
│ 2            │  ← 费用
│              │
│    插画       │
│              │
│ 穿透弹        │  ← 名称
│ 对直线敌人造成伤害 │
│ [攻击] [穿透] │
└──────────────┘
```

---

### 12.3 手牌状态

| 状态 | 表现 |
|---|---|
| 默认 | 扇形排列，轻微重叠 |
| 可使用 | 边框正常或轻微发光 |
| 能量不足 | 卡牌变暗、费用标红 |
| 鼠标 / 手指悬停 | 卡牌上移、放大 |
| 选中 | 卡牌抬起，战场出现目标层 |
| 拖拽中 | 卡牌脱离手牌，跟随手指 |
| 释放成功 | 卡牌飞向目标或进入弃牌堆 |
| 禁用 | 灰化并显示原因 |

---

### 12.4 推荐初始手牌数量

建议默认：

```text
4–5 张
```

原因：

- 竖屏空间有限
- 5 张以内比较容易保持卡牌可读性
- 与《杀戮尖塔》类手牌体验接近
- 方便做扇形展开和拖拽操作

---

## 13. 弃牌区

线框标注：

```text
弃牌
```

位置：底部左侧。

需要区分两个概念：

| 概念 | 含义 | 建议文案 |
|---|---|---|
| 查看弃牌堆 | 查看已经使用或丢弃的卡 | 弃牌堆 |
| 主动弃牌操作 | 主动丢弃当前手牌 | 弃牌 |

如果采用卡牌构筑机制，建议底部左侧显示：

```text
弃牌堆 12
```

点击后打开弃牌堆查看弹窗。

---

## 14. 牌堆区

线框标注：

```text
牌堆
```

位置：底部右侧。

功能：

- 显示当前抽牌堆剩余数量
- 玩家可点击查看牌库内容
- 牌堆为空时触发洗牌逻辑

建议表现：

```text
牌堆 18
```

点击后可查看：

- 当前牌库剩余卡牌
- 即将抽到的卡牌，如果玩法允许
- 已经消耗或移除的卡牌
- 洗牌规则说明

底部经典卡牌布局建议：

```text
[弃牌堆]     [当前手牌]     [牌堆]
```

## 15. 信息层级

### 一级信息：必须一直清楚

这些信息直接影响当前操作和生死判断。

- 当前敌人位置
- 城墙血量
- 当前能量
- 当前手牌
- 波次进度

---

### 二级信息：需要时查看

这些信息重要，但不是每次操作都需要立即读取。

- 对局时间
- 局内等级
- 人物能量
- 枪械子弹数
- 牌堆数量
- 弃牌堆数量

---

### 三级信息：辅助提示

这些信息用于增强反馈，不应长期遮挡核心区域。

- 伤害数字
- 状态图标
- 推荐落点
- 危险警告
- 经验变化
- 掉落信息
- 暴击 / 连击提示

---

## 16. 关键交互流程

### 16.1 使用手牌释放技能

```text
玩家查看当前能量
↓
选择一张可用手牌
↓
```

## 18. 视觉设计建议

### 18.1 整体风格

建议方向：

- 魔幻废墟
- 教堂广场
- 石质城墙
- 卡牌战斗 UI
- 角色站桩防守
- 战斗技能表现清晰

关键词：

```text
竖屏、塔防、防线、手牌、角色、城墙、波次、生存、局内成长
```

---

### 18.2 战斗场地

建议使用：

- 石板地面
- 中央道路
- 两侧废墟 / 植被
- 敌人从上向下推进
- 弹道从下向上发射
- 城墙位于下方作为视觉阻隔

---

### 18.3 手牌区

必须避免：

```text
固定四宫格技能栏
```

建议采用：

```text
扇形手牌
轻微重叠
中心卡牌略高
选中卡牌上浮
费用清晰
卡名清晰
```

参考体验：

- 《杀戮尖塔》
- 《吸血鬼爬行者》
- 卡牌战斗游戏的手牌区

## 19. 当前线框需要确认的问题

在进入高保真设计前，建议确认以下规则：

| 问题 | 建议方向 |
|---|---|
| 人物是否在战场中显示？ | 建议显示在城墙上方，底部只放状态 |
| 当前能量和人物能量是否分开？ | 建议分开，当前能量用于打牌，人物能量用于大招 |
| 枪械子弹数是否影响卡牌？ | 如果影响，应绑定人物状态或武器系统 |
| 默认手牌数量是多少？ | 建议 4–5 张 |
| 是否有回合制？ | 如果是实时防守，应避免出现“对方回合”类信息 |
| 牌堆 / 弃牌是否可点击查看？ | 建议可查看，符合卡牌构筑预期 |
| 局内等级升级后发生什么？ | 需要明确是抽卡、选天赋、升属性还是强化卡牌 |
| 城墙血条是否唯一失败条件？ | 若还有人物死亡条件，需要另设人物 HP |

## 20. 推荐最终布局方案

基于当前线框，推荐后续按以下结构推进视觉稿：

```text
顶部：
暂停 / 时间 / 关卡名 / 波次

顶部下方：
局内等级经验条

中部：
战斗场地
敌人、弹道、技能范围、子弹数

中下：
城墙与人物实体

底部上层：
城墙血条 / 人物头像 / 人物姓名 / 当前能量 / 大招能量

底部核心：
扇形手牌

底部两侧：
弃牌堆 / 牌堆
```

该结构能突出三件事：

1. **上方是战斗压力。**  
2. **中下方是守线目标。**  
3. **底部是卡牌决策。**

这比传统塔防技能栏更符合当前项目方向。
