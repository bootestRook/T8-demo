# 项目开发规则总览

## 目标

开发一个竖屏手机玩法 demo，核心体验是：

```text
固定时间刷怪 → 自动枪械保底输出 → 即时卡牌释放 → 费用连锁倍率 → 局内升级三选一 → 3 个核心流派构筑成型
```

第一版只做 3 个核心流派：

1. 温压弹
2. 干冰弹
3. 电磁穿刺

枪械不进入卡牌系统，只作为自动攻击和三选一被动强化存在。

---

## 第一版范围

### 屏幕与平台

```text
目标平台：手机
画面方向：竖屏
逻辑基准分辨率：1080 × 1920
引擎：Godot 4.x
语言：GDScript
```

所有 UI 和战斗坐标优先按竖屏设计，不做横屏兼容。

|---|---|
| 怪物刷新 | 全部走时间点配置表 |
| 精英怪 | 暂定 3:00 出现 |

## 总体开发原则

1. **战斗规则必须数据驱动。** 卡牌、升级、怪物、波次、数值都从配置表读取。
2. **不要为每张卡牌写一个独立脚本。** 卡牌应由通用执行器解析效果配置。
3. **枪械和卡牌系统分离。** 枪械是自动输出，卡牌是即时技能。
4. **先做可玩闭环，再扩表现。** 命中特效、屏幕震动、音效后置。
5. **第一版不做复杂状态系统。** 只实现词条需要的燃烧、冻结、冻伤、麻痹、减速、击退等最小功能。
