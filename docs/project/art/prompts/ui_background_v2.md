# ui_background_v2 Prompt

Use case: stylized-concept

Asset type: standalone vertical interface backdrop for a Godot 2D sci-fi card defense game. This is a real background asset, not a screenshot mockup.

Primary request: Create one 1080x1920 portrait background image that can replace the pure black empty UI/background areas around the existing game screen. It must be suitable for compositing behind an existing battlefield and HUD.

Scene/backdrop: a dark sci-fi ruin ambience surrounding a fortified academy plaza, with distant gothic-industrial silhouettes, broken metal walls, cables, vents, faint machinery, rubble, and subtle cyan energy conduits around the outer edges.

Composition/framing:
- Full portrait 1080x1920.
- The center 70% of the image must be dark, soft, low-contrast, and low-detail so existing battlefield art, monsters, UI bars, cards, Chinese text, and hero art stay readable when placed over it.
- Put the strongest details near the far left edge, far right edge, very top, and lower side margins.
- Bottom card area should contain dim industrial floor/side machinery texture, but no bright shapes behind text.
- No central road, no battlefield lane, no defense wall, no staging platform, no repeated card table, no complete gameplay scene.

Style/medium: polished 2D game background, semi-realistic hand-painted sci-fi industrial environment, cohesive with dark graphite metal, cracked concrete, cyan technology lights, sparse amber hazard lights, and current mobile tower-defense card-game art.

Lighting/mood: restrained dark tactical atmosphere, readable edge silhouettes, very soft vignette, no foggy blank black fill.

Color palette: charcoal, slate gray, cold concrete, muted teal/cyan glow, sparse amber pin lights. Avoid saturated purple gradients, pure black flat emptiness, bright blue areas, beige parchment, or warm fantasy tones.

Strict constraints:
- No characters.
- No monsters.
- No player.
- No cards.
- No HUD.
- No health bars.
- No numbers.
- No readable text.
- No icons.
- No logos or watermarks.
- No battlefield lane or game-board composition.
- The output must be an opaque standalone background image, not a screenshot preview.
