# UI 与 HUD

## HUD 信息

- 目标：显示当前内容单元目标和玩家下一步。
- 反馈：显示得分、状态、倒计时、生命、资源或当前系统关键状态。
- 结算：显示胜利、失败、本轮表现和重开/下一关入口。

## 战斗 UI 原型

- 基准分辨率：`1080x1920`，竖屏移动端；Web/本地包使用 `canvas_items` 拉伸模式，避免浏览器把整张 1080x1920 viewport 位图缩放到 720x1280 时造成 HUD 和卡牌文本发糊。
- 当前实现：`scenes/Game.tscn` 中维护编辑器可见的战斗 HUD 骨架，`src/ui/Hud.gd` 负责运行时数值刷新和基础交互。
- 顶部区域：暂停、对局时间、关卡名、波次、局内等级经验条。
- 战场区域：目标提示、推荐落点、弹药信息；底层由 `scenes/Game.gd` 绘制程序化战斗场地占位。
- 防线区域：城墙、防线耐久条、角色头像/姓名、当前出牌能量、人物能量条。
- 手牌区域：底部扇形 5 张手牌，左右分别是弃牌堆和牌堆按钮；当前支持按能量显示可用/不可用状态。
- 补牌反馈：`src/ui/HudDrawCardFx.gd` 只读 HUD 快照，检测新增手牌后从牌堆位置生成临时背面卡，沿弧线飞入目标手牌槽，落地翻面并触发目标手牌短促脉冲；牌堆按钮同步做数量变化弹动。该动效不修改 `PrototypeState.gd` 的抽牌、弃牌和洗牌规则。
- 获卡反馈：`src/ui/HudAcquireCardFx.gd` 消费 `PrototypeState.gd` 快照中的 `card_acquire_event`。新卡奖励结算后，如果手牌未达到 `hand_limit` 且牌堆/弃牌堆都没有可补牌来源，卡牌从奖励选择源飞入新增手牌槽，目标槽在落地前隐藏并在落地后发光；如果手牌已满，或手牌未满但已有牌可补充，新卡缩小飞向右侧牌堆按钮并触发牌堆脉冲。该动效只表现 `new_card` 奖励落点，不改变奖励、抽牌或牌堆规则。
- 连锁费用提示：`CardSnapshotBuilder.gd` 标记每张手牌是否能按下一费用维持连锁，`HudCardWidgets.gd` / `HudCardVisuals.gd` 只在“可续链、能量足够、非冷却且当前可出牌”时让费用徽章持续金色光晕和珍珠闪烁。
- 弹窗遮罩：暂停层和升级奖励层使用统一的全屏暗色 backdrop，层级位于战场与手牌之上、弹窗内容之下；背景卡牌特效在两个界面中都应被同等压暗，避免一个界面透出卡面颜色、另一个界面看不到颜色。
- 主界面入口：进入游戏先显示独立 `scenes/ui/MainMenuScreen.tscn`，提供“通关状态”、左右关卡切换和“开始游戏”按钮；开始后由 `scenes/Game.gd` 调用 `ContentUnits.load_combat_configs(level_id)` 并重置本局进入战斗。
- 通关与解锁：关卡挑战成功定义为击杀全部怪物且城墙血量 `> 0`。通关状态显示为“未通关”“已通关”“完美通过”；城墙满血通关记为“完美通过”，非满血但大于 0 记为“已通关”。只有通过当前关卡后才解锁下一关；未通过时主界面不得预览或跳转到后续关卡。
- 暂停入口：暂停按钮打开全屏半透明暂停层，提供“退出”和“继续战斗”。退出返回主界面，继续战斗恢复本局。
- 素材状态：当前为程序化 UI 原型和占位绘制，不计入正式 UI 美术素材完成。
- 视觉证据：`reports/screenshots/20260616-114220-native/running.png`、`reports/screenshots/20260616-battle-ui-browser/running-375x832.png`。

## UI 素材

- 真实首版至少接入 HUD 图标、按钮、面板、进度条或状态徽章之一。
- 没有 UI 源图时，AI 生成 UI sheet 或独立 UI sprite 后放入 `assets/ui/`。
- UI 必须遵循当前风格指南，不能和角色、怪物、场景使用割裂的材质、描边、色彩或比例。

## 卡牌组件接口

- 静态皮肤：`assets/ui/cards/card_frame_v1.png`，源图规格为 `900x1600`，运行时基准尺寸为 `150x266.667`，只包含浅色卡框、费用徽章壳、棕色标题面板、图片区齿轮底座和下方效果面板，不烘入任何具体卡牌数值或文案。
- 槽位坐标以 `900x1600` 源图为准：
  - 费用框：`(47, 46) - (193, 192)`；费用数字不占满外圈，运行时使用内圈安全文本框 `(62.4, 59.4) - (176.4, 173.4)`，白色粗体感、居中显示，禁止按能量状态改成蓝色/粉色。
  - 标题框：`(190, 36) - (852, 166)`；标题文字 bbox 的上下中线必须对齐费用框中线 `y=119`。
  - 图片/icon 框：中心 `450,540`，建议 icon 尺寸 `560x560`。
  - 效果框：`(52, 930) - (848, 1465)`；效果文字使用内部 padding 后的区域 `(94, 968) - (806, 1427)`，顶端居中显示，禁止压到图片区。
- 图片回退纹理：`assets/ui/cards/card_art_fallback_v1.png`，仅当卡牌配置提供了 `image_path` 但资源路径不可用时作为回退；没有 `image_path` 时保留卡框底图的空图片区。
- 配置字段：
  - `cost`：卡牌费用，显示到左上费用徽章。
  - `name`：卡牌名字，显示到顶部棕色标题牌，白色粗体，和费用徽章共用上下中线。
  - `image_path`：卡牌图片资源路径，例如 `res://assets/sprites/cards/fireball.png`，运行时加载到图片区；未配置时按 `school` 尝试加载已确认的流派通用 icon。
  - `effect`：卡牌效果文本；兼容旧字段 `desc`。
  - `school`：卡牌核心流派归属；当前仅映射四个通用 icon，不再显示底部流派文字铭牌。
- 已接入流派 icon：
  - `温压弹`：`assets/ui/icons_Skill/icon_keji_wenyadan.png`。
  - `电磁穿刺`：`assets/ui/icons_Skill/icon_keji_diancichuanci.png`。
  - `干冰弹`：`assets/ui/icons_Skill/icon_keji_ganbingdan.png`。
  - `枪械`：`assets/ui/icons_Skill/icon_keji_qiang.png`。
- 其他流派或通用卡 icon 暂不接入，后续确认素材后再映射。
- 兼容字段：`type` 作为小字副标签保留；`art` / `art_slot` 只在图片资源加载失败时作为文字占位。

## 能量上限锁定提示

- 资源：`assets/ui/icons/limit_lock.png`。
- 规则：当 `energy_max` 小于完整上限段数时，在未开启的上限区间中央显示锁图标；当 `energy_max` 达到完整上限时隐藏。
- 接入：`src/ui/Hud.gd` 根据战斗快照中的 `energy_max` 动态更新锁图标位置。

## 验收

- 浏览器内能清楚看到目标、反馈、结算和下一步。
- UI 文案服务玩法，不解释工程结构。
