# 游戏设计指南

## 目标

这个文档是 AI 拆解玩法的主入口。它同时覆盖轻量和中度/混合玩法，避免把用户创意压成“只能做最小演示”的单一流程。

机器可读蓝图在 `spec/gameplay_blueprints.json`。如果用户提出肉鸽、搜打撤、塔防、模拟经营、RTS、自走棋、养成、背包、暗黑刷装、跑酷、LD 关卡设计或多玩法融合，必须把激活蓝图写入 `docs/game-concept.md`。

```markdown
## 玩法蓝图

- `roguelite`
- `inventory_backpack`
- `loot_arpg`
```

## 设计原则

- 先确定玩家每 10 秒重复做什么，再决定系统。
- 首版基础闭环必须包含目标、操作、反馈、结束、重开，但它只是完整首版的一部分。
- 新建真实游戏默认做完整首版：围绕用户创意实现核心循环、必要系统、至少 3 个精细内容单元或系统阶段、结算、进度和基础反馈。
- 完整首版先保证第 1 个内容单元完整，再补齐后续内容单元、系统阶段或挑战差异；如果用户创意依赖背包、经济、联网、编辑器、关卡构筑等系统，应把它们作为首版系统拆清边界后实现。
- 中度/混合玩法可以激活多个蓝图；必须写清主循环、系统关系、优先级、数据边界和验收标准，避免系统之间互相抢职责。
- ECS、大型状态机、多场景菜单、复杂资源管线、经济系统或编辑器工具链可以按玩法目标引入；引入前必须说明玩家价值、维护边界、运行时证据和质量门禁。
- 所有数值和系统边界写入 `docs/game-concept.md`，不要只留在对话里。

## 输出格式

AI 拆解玩法时输出这些字段：

- 一句话目标
- 核心操作
- 立即反馈
- 一轮结束
- 激活蓝图
- 本轮只实现
- 首版内容单元
- 本轮美术素材
- 系统边界
- 验收清单

## 快速玩法内容单元

| 类型 | 一句话目标 | 核心操作 | 首轮重点 |
|---|---|---|---|
| 躲避生存 | 在限定时间内躲开危险并获得分数或资源 | WASD/方向键移动，触屏拖动 | 生命、倒计时、敌人、拾取、重开 |
| 点击收集 | 在限定时间内点击目标获得分数，避开惩罚目标 | 鼠标/触屏点击 | 目标生成、误点惩罚、连击、倒计时 |
| 拖拽合成 | 拖动物品，把相同或指定组合合成为目标 | 拖拽物品到槽位或另一个物品 | 格子、合成规则、空间压力、目标等级 |
| 弹射射击 | 瞄准并发射弹体，在有限次数内击中目标 | 按住拖动瞄准，松开发射 | 瞄准线、弹药、命中爆裂、目标数 |
| 平台跳跃 | 移动和跳跃，通过平台到达终点或收集关键物 | 左右移动和跳跃 | 地面/平台、掉落失败、拾取、终点 |

## 完整首版体验底线

| 品类 | 首版必须包含 |
|---|---|
| 搜打撤 | 搜刮、风险升高、撤离点、死亡损失、至少 1 个高价值诱饵。 |
| 射击/动作 | 受击反馈、死亡反馈、弹药/冷却/位置压力、至少 2 种敌人或威胁。 |
| 生存 | 刷怪或威胁阶段、难度升级、逃生/防守/坚持目标、失败结算。 |
| 平台/跑酷 | 3 段空间节奏、障碍或平台变化、掉落/碰撞失败、终点或距离目标。 |
| 塔防 | 路线、防守目标、至少 3 波节奏、至少 2 类敌人或波次组合。 |
| 经营/养成 | 资源生产、消耗压力、阶段目标、完成反馈和下一阶段目标。 |
| 背包/构筑 | 物品选择、空间/容量压力、至少 3 个物品或组合、结算影响下一轮。 |

动态品类的首版不能只用静态图：玩家和主要威胁/目标至少有 1 组运行时动画证据；优先做 walk、attack/shoot、hit/death 中对当前玩法最关键的 1-2 个动作。

复刻参考游戏时，先拆：

1. 玩家最常做的动作。
2. 操作后最重要的反馈。
3. 最短一轮的成功或失败条件。
4. 参考里的复杂系统哪些本轮实现、分阶段实现或需要先写清边界。

## 实施顺序

1. 更新 `docs/game-concept.md` 的玩法蓝图、玩法不变量、启用系统和系统边界。
2. 写清首版 3 个内容单元，以及每个内容单元的目标、差异和美术素材需求。
3. 用 `src/game/PrototypeState.gd` 写最小规则状态。
4. 用 `scenes/Game.gd` 接输入、对象、碰撞或格子交互。
5. 用 `src/ui/Hud.gd` 显示目标、资源、结算和下一步。
6. 中度系统按需新增单一职责模块，名称优先使用蓝图推荐模块。
7. 接入 `ContentUnits.gd`、`ProgressStore.gd`、`FeedbackDirector.gd`、`AudioDirector.gd` 或其他系统模块时，先保证第一个内容单元完整可玩，再补后续精细内容单元、系统阶段或挑战差异。

## 混合玩法拆解示例

肉鸽 + 背包 + 暗黑刷装：

- 主循环：战斗后获得物品，放进 3x3 背包，下一波自动生效。
- 本轮实现：按目标拆出物品、奖励选择、背包格子规则、词条、装备等级、商店或局外天赋中的必要子集。
- 系统边界：写清哪些规则影响战斗、哪些规则影响局外成长、哪些规则只影响 UI 展示。
- Review 重点：奖励是否能进入背包、背包是否改变下一波战斗、失败是否结算。
