这个文件是兼容旧脚本和 review 的当前游戏事实源摘要；详细长期文档以 `docs/project/` 为准。

- `vertical-card-defense-demo`

## 一句话目标

- 这是一个竖屏 2D 卡牌连锁 + 塔防/幸存者战斗 Demo：玩家守住防线，在自动枪械射击和手牌出牌之间做资源取舍，通过连续费用连锁、同流派继承和升级三选一抵御 20 波怪物。

- 原始设定：当前轮未提供新的长设定。
- 分层项目文档：`docs/project/`
- 运行时事实源：`project.godot`、`scenes/Game.tscn`、`scenes/Game.gd`、`src/game/PrototypeState.gd`、`assets/data/combat/*.json`、`assets/data/cards/card_configs.json`

## 平台与目标层级

- 平台：Godot 4，Web/Windows，竖屏移动端画布 `1080x1920`。
- 目标层级：可试玩战斗 Demo，不是空脚手架。

## 核心玩法

- 核心操作：选择关卡开始战斗，战斗中点击/拖拽手牌出牌，必要时弃牌重组手牌。
- 立即反馈：自动枪械射击、卡牌投射物/区域效果、伤害飘字、手牌飞行动效、连锁提示、护盾/血量/能量/HUD 更新。
- 一轮结束：清完当前关卡波次且防线未毁为胜利；防线生命归零为失败。
- 重开方式：结算重开按钮、战斗中 `R` 或 `ui_accept` 重置当前战斗。

## 玩法不变量

- 应用启动先显示主菜单；菜单阶段不初始化完整战斗局。
- 选择关卡后才加载该关卡配置并 `PrototypeState.reset()`。
- 防线生命由 `PrototypeState.DEFAULT_WALL_HP = 3000` 控制，不由关卡表写入。
- 怪物从竖屏战场上方向防线推进，波次来自 `assets/data/combat/waves.json`。
- 费用连续递增才续连锁；弃牌断链；同流派后置卡继承前置正面效果。
- 卡牌、怪物、关卡和波次运行时配置必须通过数据校验脚本检查缺字段、非法数值和跨表引用。
- 运行时代码不得直接加载 `references/`。

## 禁止污染项

- 本轮不新增玩法、卡牌、怪物、关卡、数值字段、结算字段或美术资源。
- 不把防御塔建造、背包、经济、联网、排行榜等未确认系统写入当前 Demo 范围。
- 新增系统必须写清玩家价值、职责边界、数据归属、UI/素材需求和验证方式。

## 美术方向

- 当前锁定风格：科幻防线 + 教堂/城市战场方向，竖屏移动端可读优先。
- 素材落地状态：已有正式运行时素材，包括战场、防线、玩家、怪物、Boss、卡牌框、流派 icon、HUD 图标和背景。
- 默认素材路径：`assets/generated/runtime/`、`assets/sprites/`、`assets/ui/`。
- 概念图裁剪块、`style_candidates/` 候选图、`references/` 参考图和临时 workspace 文件不得直接作为运行时素材。

## 首版范围

- 当前可玩范围：主菜单关卡选择、通关解锁、5 个关卡配置、每关 20 波、手牌/牌堆/弃牌堆、费用连锁、升级三选一、胜负结算和重开。
- 当前不包含：防御塔建造、主动移动、背包经济、联网、排行榜、额外结算指标。

## 首版内容单元

- 关卡 1：1-1 教堂广场，入门节奏，血量系数 0.5，基础小怪与快速怪，差异是低风险教学和连锁熟悉。
- 关卡 2：1-2 教堂侧廊，血量系数 0.88，差异是更紧的波次与更高生存压力。
- 关卡 3：1-3 教堂中庭，攻击系数 1.1、血量系数 1.25，差异是更高耐久和目标优先级压力。
- 关卡 4：1-4 钟楼前线，攻击系数 1.3、血量系数 1.63，差异是精英压力和资源取舍更强。
- 关卡 5：1-5 教堂广场决战，攻击系数 1.5、血量系数 2.13，第 16 波 Boss，差异是 Boss 检查点和高峰压力。

## 品类说明

- 塔防侧重点：路线、波次、基地生命/防线生命；当前不做防御塔建造。
- 关卡设计侧重点：关卡目标、空间、节奏和通关检查点；当前检查点体现为主菜单通关状态与下一关解锁。

## 系统边界

- `PrototypeState.gd`：战斗局状态、规则编排、public API 和 snapshot。
- `Game.gd`：应用启动、主菜单/战斗切换、输入转发和表现绘制。
- `Hud.gd`：节点引用、生命周期入口、信号转发和少量 orchestration；纯查询、纯流程、纯表现刷新下沉到 helper。
- `ContentUnits.gd`：关卡、波次、怪物配置读取。
- `scripts/validate_data_configs.py`：JSON 配置字段、数值和跨表引用校验。

## 验收标准

- `python scripts/validate_data_configs.py` 通过。
- `python scripts/generate_project_map.py` 通过。
- `python scripts/godot_headless_check.py` 通过。
- `python scripts/godot_runtime_log_check.py` 通过。
- 如环境可用，`python scripts/ai_review.py --strict` 通过。
- 浏览器或原生运行中能完成主菜单、选择关卡、开始战斗、暂停/退出、胜负结算和重开。
