# 游戏设定卡

这个文件是当前游戏概念的单一事实源。当前模板处于空脚手架状态，不预置任何玩法、角色、关卡、数值或运行时美术。

## 概念ID

- `starter-template`

## 玩法蓝图

- `starter_template`

## 一句话目标

- 提供一个无内置 Demo 的 Godot 4 AI 游戏脚手架；用户输入 `init` 后再创建新的游戏概念、玩法蓝图、美术方向和完整首版内容单元。

## 平台与目标层级

- 平台：浏览器 Web。
- 目标层级：脚手架启动页和工具链验证，不代表任何具体游戏。

## 当前状态

- 无默认主循环。
- 无默认玩家、敌人、目标、关卡、胜负、分数或素材。
- 无内置游戏。
- 主场景只显示中性启动提示，避免 AI 误沿用模板内置游戏。

## 玩法不变量

- 新游戏必须通过 `init` 或明确需求创建新的概念ID。
- 新游戏必须写入一句话目标、核心操作、立即反馈、胜负/结算、玩法不变量、美术方向、首版内容单元或系统阶段、本轮美术素材计划和系统边界。
- 运行时代码不得直接加载 `references/`。
- 不得从旧 demo 继承角色、数值、胜负条件或素材路径。

## 禁止污染项

- 模板不得预置内置 Demo 玩法。
- 模板不得预置运行时 demo 素材。
- 模板不得把示例玩法当作默认验收标准。
- 参考资料只能进入 `references/`，运行时素材必须进入 `assets/` 或 `addons/`。

## 美术方向

- 当前锁定风格：未选择。
- 素材落地状态：空脚手架，无运行时美术。
- 新游戏创建后，先使用 `asset-prompt-spec` 确认用途、尺寸、数量、背景和落地路径，再使用 `aistudio-media-generation` 生成或导入素材。
- 概念图只作为风格锚点；风格锁定后必须生成同源运行时视觉包，HUD、角色、怪物/目标、场景和 VFX 不得直接裁剪候选图或各自临时拼贴。

## 首版范围

## 首版内容单元

- [ ] 暂无。新游戏 init 后再写入至少 3 个首版内容单元、挑战或系统阶段及差异。

## 首版基础验证

- [x] Godot 主场景可启动。
- [x] 不预置内置游戏。
- [x] AI workflow 能引导用户创建新概念。
- [x] AI review 能识别空脚手架状态。

## 本轮美术素材计划

- [ ] 暂无。新游戏 init 后再围绕首版内容单元和系统阶段列出最高收益静态素材；动态品类补关键角色小批量动作帧，并补 `docs/project/art/style-guide.md` 或等效风格规则。

## 完整首版必做

- [ ] 新游戏创建后默认至少 3 个有差异内容单元、挑战或系统阶段。
- [ ] 有 30 秒内阶段变化、奖励诱惑、威胁升级或目标转折。
- [ ] 有结算反馈和简单进度。
- [ ] 动态品类有关键角色动画证据。
- [ ] HUD、角色、威胁/目标、场景和 VFX 来自同一视觉规范和运行时素材包。

## 后续增强

- [ ] 新游戏创建后扩展到 4-5 个内容单元。
- [ ] 接入真实素材、UI sprite、音效和 VFX。

## 系统边界

- 内置示例游戏。
- 账号、联网、付费、背包、经济系统、排行榜、关卡编辑器等能力不再默认排除；新游戏按用户创意和验收风险决定是否进入首版。
- 新增系统必须写清玩家价值、职责边界、数据归属、UI/素材需求和验证方式。

- `python scripts/gameplay_logic_review.py` 通过。
- `python scripts/art_pipeline_review.py` 通过。
- `python scripts/experience_design_review.py` 通过。
- `python scripts/godot_quality_tools.py --json` 通过。
- `python scripts/godot_headless_check.py` 通过。
- `python scripts/export_web.py --json` 通过。
- `python scripts/experience_check.py --strict` 通过。
- `python scripts/ai_review.py --strict` 通过。
- 全文扫描中，旧 demo 词只能出现在历史/禁止沿用语境中，不能作为默认玩法或运行时依赖出现。
