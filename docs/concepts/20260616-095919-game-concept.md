# 游戏设定卡

## 概念ID

- `vertical-defense-card-v1`

## 玩法蓝图

- `hybrid`
- `roguelite`

## 一句话目标

- 竖屏手机 Web 防线卡牌游戏：固定时间刷怪，自动枪械保底输出，玩家用能量即时出牌，通过费用连锁倍率和升级三选一把温压弹、干冰弹、电磁穿刺三种流派构筑成型，守住城墙并击败 5 分钟 Boss。

## 设定来源

- 本轮策划原文：`策划文档/00_项目总览/00_项目总览.md` 至 `策划文档/06_里程碑验收/09_里程碑验收清单.md`
- 分层项目文档：`docs/project/`

## 平台与目标层级

- 平台：手机 Web
- 画面方向：竖屏
- 逻辑基准分辨率：1080 x 1920
- 目标层级：完整首版可试玩 demo

## 核心玩法

- 主循环：时间轴刷怪 -> 自动枪械近墙优先射击 -> 玩家消耗能量点击 4 手牌 -> 连锁倍率提高技能伤害 -> 击杀升级三选一 -> 3:00 精英和 5:00 Boss 检定构筑。
- 核心操作：点击 4 张手牌释放温压弹、干冰弹或电磁穿刺；必要时点击弃牌重抽中断连锁。
- 成功条件：Boss 死亡进入胜利结算。
- 失败条件：城墙生命归零进入失败结算。
- 重开方式：胜负后点击“重新开始”、按 R 或按确认键。

## 玩法不变量

- 能量限制必须迫使玩家做出出牌时机和卡牌优先级选择。
- 连锁倍率必须奖励费用递增的出牌顺序，弃牌必须中断连锁。
- 随机升级三选一必须改变一局内构筑方向。
- 一局结束必须给出胜利或失败结算，并能不刷新页面重开。

## 禁止污染项

- 不沿用上一版末日僵尸、背包格子、装备掉落或 loot ARPG 目标。
- 不把 `references/`、风格候选图或临时工作区文件直接接入运行时。
- 程序化占位不能宣称为正式美术素材。

## 模块边界

- CoreLoopContract：`PrototypeState.gd` 统一维护单局状态、胜负、能量、手牌、连锁、升级和怪物推进。
- SystemBridge：`Game.gd` 只负责输入、绘制和表现同步；`Hud.gd` 只负责 HUD、手牌按钮、升级按钮和结算入口。
- ScopeCut：第一版不做背包格子、装备掉落、联网账号、剧情养成和复杂状态机。
- ReviewMatrix：首版验收以 5 分钟完整一局、3 个阶段差异、三流派可用、Web 导出和自动体验检查为准。
- 副系统：能量、卡牌、连锁、同名冷却、自动枪械、升级草案、进度保存。
- 优先级：先保证完整一局和三流派可玩，再补正式素材、CSV ConfigDB 和更多词条。
- 首版裁剪：复杂背包、装备掉落、剧情、账号、排行榜和正式资产库后置。
- RunState：单局内等级、经验、能量上限、手牌、牌库、弃牌堆和已选升级均由 `PrototypeState.gd` 管理。
- RewardDraft：升级暂停并生成三选一，选项覆盖获得新卡、强化已有卡、枪械、生存、能量、核心技能强化。
- MetaProgress：胜负后记录最佳分数与 Boss 完成标记到 `ProgressStore.gd`。
- ContentUnits：`ContentUnits.gd` 保存 3 个阶段、卡牌配置、升级配置、波次配置和怪物配置。
- 随机：升级池和牌库洗牌每局随机。
- 一局结束：Boss 死亡或城墙归零立即结算。
- 局外：本地记录最佳分数和 Boss 完成标记，后续可扩展局外成长。

## 首版内容单元

- 内容单元 1：基础防线。差异：小怪和快怪分批推进，18 秒出现快怪，38 秒数量增加；目标是教学自动枪械、能量回复、4 手牌和基础出牌。
- 内容单元 2：精英压迫。差异：重甲怪加入，180 秒精英出现，敌人血量和城墙压力明显上升；目标是检验升级三选一、枪械强化、控制和爆发取舍。
- 内容单元 3：Boss 检定。差异：300 秒 Boss 出现，Boss 高血量和高城墙伤害要求至少一个流派成型；目标是检验连锁倍率、核心技能强化和自动枪械保底。

## 三个核心流派

- 温压弹：冲击、爆炸、击退、燃烧，适合清密集怪和缓解近墙压力。
- 干冰弹：穿透、击退、冻结、齐射，适合直线清怪和控制精英。
- 电磁穿刺：单体穿刺、麻痹、爆炸、矩阵、裂变，适合点杀高威胁和制造减速区域。

## 美术方向

- 当前风格：高读图竖屏科幻防线，深色战场、亮色技能弹道、清晰城墙和怪物轮廓。
- 素材落地状态：程序化占位。首版用 `Game.gd` 绘制角色、怪物、弹道、VFX 和 HUD 面板；不是正式美术素材。
- 运行时 VFX 依赖：补齐 `addons/vfx_library/assets/effect/` 中缺失的粒子贴图，供内置 VFX addon 正常加载。
- 后续优先项：生成同源玩家炮台、四类怪物、三流派弹道 spritesheet、HUD 卡牌面板和按钮素材，登记到 `docs/project/art/asset-manifest.json`。

## 本轮美术素材计划

- 当前首版使用程序化占位：玩家炮台、城墙、怪物、Boss、弹道、VFX 和 HUD 均由运行时代码绘制。
- 已补运行时支持贴图：内置 VFX addon 的粒子示例改为内嵌渐变贴图，避免加载缺失资源。
- 下一轮正式素材：同源玩家炮台、基础怪/快怪/重甲怪/精英/Boss、温压/干冰/电磁弹道、卡牌 UI 和按钮面板。

## 验收标准

- 5 分钟内有小怪、快怪、重甲怪、精英、Boss 阶段变化。
- 玩家 30 秒内能看见能量增长、手牌可点、怪物推进、自动枪械开火和技能反馈。
- 4 手牌、弃牌冷却、同名卡冷却、费用连锁、升级三选一可用。
- 胜利、失败、分数、击杀、重开和本地最佳分数记录可用。
- `python scripts/export_web.py --json` 能导出 Web 版本。
